# Desarrollo Web: Proyecto UniMarket

## 1. Diferencia entre Página Web y Aplicación Web

**UniMarket** se define como una **aplicación web**, ya que permite el registro de usuarios, la publicación de productos y servicios, así como la gestión dinámica de información.

### Comparación con UniMarket

| Aspecto | Página Web | Aplicación Web | UniMarket |
|--------|------------|----------------|-----------|
| **Interacción** | Informativa, solo lectura | Interactiva y dinámica | Alta interacción |
| **Usuarios** | Lectores | Usuarios registrados | Estudiantes compradores y vendedores |
| **Datos** | Contenido estático | Datos en tiempo real | Productos, servicios y mensajes |
| **Funcionalidad** | Mostrar información | Ejecutar procesos | Publicar, comprar y contactar |
| **Ejemplo** | Blogs, sitios institucionales | Marketplaces y redes sociales | Marketplace estudiantil |


## 2. ¿Qué es una Aplicación Web?

Una aplicación web es un software accesible desde un navegador que permite realizar acciones, procesar información y almacenar datos.

En el contexto del **Marketplace UniMarket**, permite:

- Crear cuentas de usuario.
- Publicar y administrar productos o servicios.
- Buscar y filtrar información de manera eficiente.
- Interactuar directamente entre estudiantes.
- Almacenar datos en una base de datos centralizada.


## 3. Ejemplos Reales de Aplicaciones Web Profesionales

| Aplicación Web | Función Principal | Relación con UniMarket |
|---------------|------------------|------------------------|
| **Mercado Libre** | Compra y venta | Modelo de marketplace |
| **Facebook Marketplace** | Venta local | Venta directa entre usuarios |
| **Amazon** | Comercio electrónico | Gestión de productos |
| **Uber** | Plataforma de servicios | Intermediación usuario-servicio |
| **Fiverr** | Servicios freelance | Venta de servicios |


## 4. ¿Qué tipo de problemas se resuelven con Software?

### Problemas Generales

- Organización de grandes volúmenes de información.
- Automatización de procesos manuales.
- Comunicación instantánea entre personas.
- Acceso a servicios desde cualquier lugar.

### Problemas Específicos que resuelve UniMarket

1. **Falta de visibilidad:** Los productos y servicios de los estudiantes suelen pasar desapercibidos.
2. **Dificultad de búsqueda:** Complicación para encontrar clientes dentro de la comunidad.
3. **Comunicación desordenada:** Uso de canales informales como grupos saturados.
4. **Inseguridad:** Ausencia de un espacio centralizado exclusivo para la comunidad académica.

## 5. Arquitectura General de una Aplicación Web

| Capa | Función | Aplicación en el Proyecto |
|-----|--------|---------------------------|
| **Frontend** | Interfaz visual | UI del marketplace |
| **Backend** | Lógica y reglas | Gestión de usuarios y validaciones |
| **Base de Datos** | Almacenamiento | Usuarios, productos y mensajes |
| **Entornos** | Ciclo de vida | Desarrollo, pruebas y despliegue |

### Tecnologías sugeridas para UniMarket

- **Frontend:** HTML, CSS, JavaScript (React o Vue.js).
- **Backend:** Node.js con Express.
- **Base de datos:** MySQL, PostgreSQL o MongoDB.


## 6. Plataformas Similares

### 6.1 Facebook Marketplace

- **Similitudes:** Publicación de productos y contacto directo.
- **Diferencias:** UniMarket se enfoca exclusivamente en estudiantes y permite validación académica.

### 6.2 Fiverr

- **Similitudes:** Venta de servicios especializados y perfiles de usuario.
- **Diferencias:** Fiverr es global y profesional; UniMarket es local y estudiantil.


## Conclusión

**UniMarket** es una aplicación web diseñada para resolver problemas reales dentro de la comunidad académica. Al adaptar modelos de plataformas profesionales al contexto estudiantil, se prioriza la accesibilidad, la confianza y el apoyo entre compañeros.
