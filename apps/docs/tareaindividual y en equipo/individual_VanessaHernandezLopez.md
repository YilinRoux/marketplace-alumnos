# Investigación sobre aplicaciones web y desarrollo de software

## Diferencia entre página web y aplicación web

### Página web

Una **página web** es un sitio cuyo propósito principal es mostrar información al usuario. Generalmente es estática o con muy poca interacción y no procesa datos complejos.

**Características:**
- Contenido informativo.
- Interacción limitada.
- No requiere registro de usuarios.
- No maneja bases de datos complejas.

**Ejemplos:**
- Página oficial de una escuela.
- Sitio web de una empresa con información de servicios.
- Portafolio profesional en línea.

---

### Aplicación web

Una **aplicación web** es un sistema interactivo que funciona desde un navegador y permite a los usuarios realizar acciones específicas, gestionar información y recibir respuestas en tiempo real.

**Características:**
- Alta interacción con el usuario.
- Uso de bases de datos.
- Procesamiento de información.
- Autenticación de usuarios.
- Funciona como un sistema completo.

**Ejemplos:**
- Sistemas de ventas en línea.
- Plataformas educativas.
- Aplicaciones bancarias en línea.

---

## Ejemplos reales de aplicaciones web profesionales

Las aplicaciones web profesionales son utilizadas diariamente por millones de personas para resolver distintas necesidades.

**Ejemplos:**
- **Google Drive:** Permite crear, editar y almacenar documentos en la nube.
- **Facebook:** Red social que gestiona perfiles, publicaciones e interacciones.
- **Mercado Libre:** Plataforma de comercio electrónico para comprar y vender productos.
- **Trello:** Herramienta para la gestión de proyectos y tareas en equipo.

Estas aplicaciones destacan por su estabilidad, escalabilidad y facilidad de uso.

---

## ¿Qué tipo de problemas se resuelven con software?

El software se desarrolla para resolver problemas reales en distintos ámbitos, tales como:

- **Organización de información:** Control de clientes, productos o registros.
- **Automatización de procesos:** Facturación, control de pedidos y reportes.
- **Comunicación:** Mensajería, videollamadas y redes sociales.
- **Productividad:** Gestión del tiempo, tareas y proyectos.
- **Comercio:** Ventas en línea, pagos digitales y envíos.

El objetivo principal del software es optimizar procesos, reducir errores y ahorrar tiempo.

## Arquitectura general de aplicaciones web

Una aplicación web se divide en tres partes principales:

### Frontend
Es la parte visual con la que interactúa el usuario.

- Desarrollado con HTML, CSS y JavaScript.
- Incluye formularios, botones y vistas.
- Frameworks comunes: React, Angular, Vue.js.

### Backend
Es la parte lógica que procesa la información.

- Aplica reglas de negocio.
- Gestiona la comunicación con la base de datos.
- Lenguajes comunes: PHP, Java, Python, Node.js.

### Entornos
Son los espacios donde se ejecuta la aplicación.

- **Desarrollo:** Donde se programa y prueba el sistema.
- **Pruebas:** Donde se detectan errores antes de publicar.
- **Producción:** Donde la aplicación es utilizada por los usuarios finales.

## Análisis de 2 plataformas reales similares a una idea de aplicación web

### 1. Rappi

**Descripción:**  
Rappi es una plataforma web y móvil que permite solicitar productos de restaurantes, supermercados y farmacias.

**Funciones principales:**
- Registro de usuarios.
- Catálogo de productos.
- Pagos en línea.
- Seguimiento de pedidos.

**Problema que resuelve:**  
Facilita la compra de productos sin que el usuario tenga que desplazarse.

---

### 2. Uber Eats

**Descripción:**  
Uber Eats es una plataforma de pedidos de comida en línea que conecta usuarios, restaurantes y repartidores.

**Funciones principales:**
- Búsqueda de restaurantes.
- Personalización de pedidos.
- Pago digital.
- Historial de pedidos.

**Problema que resuelve:**  
Optimiza el proceso de pedido y entrega de alimentos de manera rápida y eficiente.

---

## Conclusión

La diferencia entre una página web y una aplicación web es clave para el desarrollo de proyectos tecnológicos. Las aplicaciones web permiten resolver problemas reales mediante sistemas interactivos que automatizan procesos y mejoran la experiencia del usuario. Analizar plataformas existentes ayuda a identificar funcionalidades importantes y buenas prácticas para el desarrollo de nuevos sistemas.
